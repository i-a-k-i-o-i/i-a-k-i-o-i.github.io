简介

AK IOI，即在国际信息学奥林匹克竞赛（IOI）中获得满分，是无数 OIer 的终极梦想。本文将系统性地阐述 AK IOI 的完整路径，涵盖从哲学思想、知识储备、实战技巧到赛前准备等全方位指导。

核心观点： AK IOI 是一个系统工程，其本质是战胜过去的自己。本教程不仅教你算法，更教你如何思考。

无论你是刚接触信息学竞赛的新手，还是有一定经验希望突破自我的选手，这篇文章都将为你提供有价值的指导。我们将从最基础的心态建设开始，逐步深入到高级算法和赛场策略。

什么是IOI？

国际信息学奥林匹克竞赛（IOI）是面向中学生的最高级别信息学竞赛，每年吸引来自全球近百个国家的顶尖选手参与。竞赛通常包含 6 道题目，需要在两天各 5 小时的比赛中解决。

1

思想准备 - AK 是一种心态

在开始学习任何算法之前，你必须先拥有 AK 者的心态。

许多选手过于关注具体的技术细节，而忽略了心态建设的重要性。实际上，在 IOI 这样的高水平竞赛中，心态往往成为决定胜负的关键因素。

重新定义“AK”

对你而言，AK 不应该是一个结果，而是一种过程。每次模拟赛/正赛的 AK，每次对难题的透彻理解，都是向最终目标迈进的一步。

实用建议： 将大目标分解为小目标。不要想着一上来就 AK IOI，而是先专注于 AK 一场模拟赛，或者将一个算法理解透彻。

拥抱失败

你将会有 99% 的时间在 WA/TLE/RE 等 Unaccepted 中度过。把这些错误信息当成是你的朋友，它们是评测姬在给你提示，而不是嘲讽。

// 调试的艺术：将错误视为学习机会
while (problem.hasBug()) {
    debug(problem);
    learnFromMistakes();
    problem.improve();
}
2

知识体系 - 建造你的武器库

光有心态是不够的，你需要实打实的“武器”。

第一层：基础内功（必须滚瓜烂熟）

  • 语言：熟练掌握 C++ 语法。
  • 数据结构： 数组、链表、栈、队列、二叉树、图（邻接表、邻接矩阵）。
  • 算法： 排序、二分查找、递归、深度优先搜索（DFS）、广度优先搜索（BFS）、动态规划（DP）基础等。

第二层：进阶神功（IOI 奖牌的分水岭）

  • 数据结构： 线段树、树状数组、并查集、哈希表、堆等。
  • 算法：
    • 图论： 最短路（Dijkstra, SPFA）、最小生成树（Kruskal, Prim）、拓扑排序、强连通分量（SCC）。
    • 动态规划： 状态压缩DP、区间DP、树形DP。
    • 数学： 数论（gcd，同余）、组合数学。

第三层：屠龙之技（AK 的钥匙）

  • 高级数据结构： 平衡树（Splay, Treap）、可持久化数据结构、树链剖分。
  • 高级算法：
    • 图论： 网络流、二分图匹配。
    • 字符串： KMP、Trie 树、后缀数组/自动机。
    • 计算几何： （IOI 常客，但难度爆炸）凸包、旋转卡壳等。
  • 思维技巧： 贪心、分治、折半搜索、Meet in the Middle。

学习路径建议： 不要试图一次性掌握所有高级算法。先扎实掌握基础，然后根据比赛需求和个人兴趣选择进阶方向。质量比数量更重要。

实战技巧 - 赛场上的生存法则

你知道所有算法，但比赛时就是写不出来？你需要这些技巧。

"暴力出奇迹"原则

永远先写一个最简单的暴力程序（Brute Force）。这有三个好处：

  1. 帮你理解题意。
  2. 提供对小数据的确信答案，用于对拍。
  3. 万一数据水，你可能就 AC 了。（虽然在 IOI 中，这是极小概率的）

对拍是神

写一个数据生成器，用你的暴力程序和"高级程序"同时跑大量随机数据，比较结果。这是发现程序漏洞最有效的方法。

// 简单的对拍脚本思路
while (true) {
    generate_test_data();
    run_brute_force();
    run_optimized_solution();
    if (results_differ()) {
        debug_and_fix();
    }
}

时间管理

简单题要稳、准、快，为难题留出时间。一道题如果卡了很久毫无头绪，果断切换下一题。

赛场黄金法则： 先确保拿到所有能拿的分数，再去挑战难题。在 IOI 中，部分分往往决定了奖牌的成色。
